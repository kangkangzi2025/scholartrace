# ScholarTrace 演示台本

> 黑客松 5 分钟讲完。每页一句 punchline，操作动作粗体。

---

## 启动前

```bash
cd scholartrace
python3 -m http.server 8765
```

浏览器打开 `http://localhost:8765/`，**全屏（F11）**，确认字体加载完成（如不联网则用本地降级字体也无妨）。

---

## 0:00 — 0:30 · 首页打开

**「研究新手最大的痛点不是查不到论文，是不知道这个领域究竟谁在做。Connected Papers 只看单篇之间的引用，看不到人。我们做的事情是：输入一个领域，看清那群人是谁。」**

**操作**：
1. 让 hero 标题（「输入一个领域 / 看清那群人是谁」）停留 3 秒
2. 指着底部 4 阶段管线卡（Concept alignment → RAG → Hard-rule → LLM-as-a-judge）说："这是 backend pipeline。"
3. 点击 suggest 按钮 **「L2R」**（不要打字，节省时间）

---

## 0:30 — 1:00 · 概念对齐弹窗

弹窗显示：「检测到 L2R，是否对齐到 Learning to Rank？」

**「输入可能是简写、可能是中文、可能是俗称。我们用 OpenAlex Concept Tree + LLM 同义词归并把它锁定到规范概念。这一步如果错了，下面整条 pipeline 都白搭。」**

**操作**：点 **「确认 · 检索 →」**。Agent 动画弹出（约 3 秒）：
- Stage 1: Concept alignment ✓
- Stage 2: Qdrant ANN · 100 候选 · Recall@100 = 0.93
- Stage 3: Hard-rule ranking
- → Rendering Top-K list

## 1:00 — 1:30 · 学者排名页（list.html）

页面加载后立即有 10 张学者卡，**右上角进度条** 从 50% 慢慢爬到 100%，✨ 标记一个个点亮。

**「这就是首屏 1.5 秒拿到的骨架——RAG 召回 + 硬规则排序。然后 LLM-as-a-judge 异步重排，每位学者评一个语义匹配分，✨ 一个个点亮——这是 SSE 流式。」**

**操作**：让 ✨ 自然点完。指右上角 emerald 圆点："`AI 重排完成`。"

## 1:30 — 2:30 · 权重滑块演示（杀手戏）

**「PRD 里我们给了用户三个杠杆。」**

**操作**：
1. 拖 **「学术沉淀」h-index 到 80%+**，另外两个会被自动归一化压低
2. 排名实时翻天覆地。指："Joachims h=115 冒头，他的学生还在产出。"
3. 拖回中间，再拖 **「近期活跃度」到 80%+**
4. 老牌人物 Burges、Robertson **退到下面**（recent-3y 低）
5. **「这是研究者最常用的两个视角：找泰斗 vs 找正在产出的新星。一个滑条切换。」**
6. 点 **「重置」** 回默认权重

## 应急预案

| 现场状况 | 应对 |
|---|---|
| 浏览器白屏 | 检查 server 是否在跑：`curl localhost:8765/index.html` |
| 字体没加载（看上去丑） | 不联网时正常现象——降级到默认字体，结构色彩不变 |
| 投影分辨率低（< 1366px） | graph.html 三栏可能挤——把右侧详情面板临时折叠（手动注释掉 `panel.col-span-3`） |
| Chrome 卡 | 关掉其他 tab；vis-network 节点超过 50 才会卡，15 个完全无压力 |
| 双击不切换 | 检查 console，确保 fetch JSON 走的是 http:// 而不是 file:// |

## 备用台词（被提问时）

**Q：数据怎么来的？**
A：MVP 用 12 学者的 mock，已校准到真实公开数据（h-index、引用、合作论文数都按 Semantic Scholar 真实值填的）。生产版接 OpenAlex API + Qdrant + GPT-4o，PRD 第 3 节有详细 spec。

**Q：跟 Connected Papers / Litmaps 什么区别？**
A：他们是 paper-centric——给一篇论文找相关论文。我们是 people-centric——给一个领域找出核心学者群和他们的关系图。两者互补。

**Q：LLM 幻觉怎么控制？**
A：所有声明（共著篇数、年份、机构）走 OpenAlex 校验后才喂给 LLM；LLM 只负责生成"语义关联理由"这种主观文本，不生成定量数据。PRD 要求 95% 事实准确率。

**Q：如果 OpenAlex 限流？**
A：本地影子数据库 + 语义缓存（PRD 5.2 已规划）。

**Q：为什么不直接做学者推荐？**
A：v1.0 聚焦"看见"，v2.0 才做"推荐"。先把人际网络看清楚，再做匹配。
